What happened:

I attempted to use the redis backend for remote_cache.

To configure it, I exported the environment variables

GF_REMOTE_CACHE_TYPE=redis
GF_REMOTE_CACHE_CONNSTR=addr=a.b.c.d:25224,db=0 

Grafana fails to start with the log

"Service init failed: incorrect redis connection string format detected for '', format is key=value,key=value"

If I set the relevant property in grafana.ini, everything works fine; the bug is only when I use the environment variable to attempt to configure the remote_cache subsystem.

What you expected to happen:

Grafana starts up and uses the redis instance on host a.b.c.d port 25224

How to reproduce it (as minimally and precisely as possible):

Attempt to configure the remote_cache with the GF_REMOTE_CACHE_CONNSTR environment variable

Anything else we need to know?:

This is almost certainly because the connstr key is not present in conf/defaults.ini so cannot be overridden by environment variables.

This is moderately infuriating.
